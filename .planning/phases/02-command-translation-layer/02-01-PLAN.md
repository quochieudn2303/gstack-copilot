# Plan 02-01: Inventory & Environment Variable Translation

**Phase:** 2 - Command Translation Layer  
**Plan:** 1 of 3  
**Requirements:** TOOL-03 (partial), groundwork for TOOL-01, TOOL-02, TOOL-04

## Objective

Inventory the actual Bash patterns used in the core 4 skills (`/review`, `/qa`, `/office-hours`, `/ship`) and implement environment variable translation as the foundation for all subsequent command translation work.

## Tasks

### Task 1: Clone and Inventory gstack Skills
**File:** `scripts/inventory.ts`

1. Clone gstack repository to temp location (or use existing if present)
2. Extract all Bash code blocks from the 4 core skills:
   - `.factory/skills/gstack-review/SKILL.md`
   - `.factory/skills/gstack-qa/SKILL.md`
   - `.factory/skills/gstack-office-hours/SKILL.md`
   - `.factory/skills/gstack-ship/SKILL.md`
3. Parse and categorize:
   - Environment variables used (`$HOME`, `$GSTACK_ROOT`, `$_ROOT`, etc.)
   - UNIX utilities called (`find`, `grep`, `date`, `wc`, `cat`, `mkdir`, `touch`, etc.)
   - Process substitution patterns (`source <(...)`)
   - Shell conditionals and tests (`[ -f ... ]`, `[[ ... ]]`)
4. Output inventory to `.planning/phases/02-command-translation-layer/INVENTORY.md`

### Task 2: Environment Variable Mapping Registry
**File:** `src/mappings/envvars.ts`

Create declarative registry for environment variable translations:

```typescript
export interface EnvVarMapping {
  bash: string;           // e.g., "$HOME"
  powershell: string;     // e.g., "$env:USERPROFILE"
  description: string;
}

export const envVarMappings: EnvVarMapping[] = [
  { bash: "$HOME", powershell: "$env:USERPROFILE", description: "User home directory" },
  { bash: "$GSTACK_ROOT", powershell: "$env:GSTACK_COPILOT_ROOT", description: "gstack-copilot installation root" },
  { bash: "$GSTACK_BIN", powershell: "$env:GSTACK_COPILOT_BIN", description: "Binary directory" },
  // ... more from inventory
];
```

### Task 3: Environment Variable Transform Stage
**File:** `src/pipeline/transforms/envvars.ts`

Implement transform function that:
1. Finds all `$VAR` patterns in content
2. Applies mappings from registry
3. Converts unmapped `$VAR` to `$env:VAR` syntax
4. Handles edge cases:
   - `${VAR}` brace syntax
   - `$VAR_WITH_UNDERSCORE` (word boundaries)
   - Escaped `\$VAR` (should not transform)

### Task 4: Integrate into Content Pipeline
**File:** `src/pipeline/content.ts` (modify)

Add environment variable transform to the content transformation chain, after path transforms but before output generation.
