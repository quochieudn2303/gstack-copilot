  <files>src/schemas/gstack.ts, src/schemas/copilot.ts, src/schemas/index.ts, tests/schemas.test.ts</files>
  <behavior>

    - Test: Valid Copilot frontmatter parses successfully
    - Test: preamble-tier validates as integer 1-4
    - Test: allowed-tools validates as array (gstack) or string (copilot)
  </behavior>
  <action>
Create Zod schemas per CONTEXT.md strict validation requirement:

1. src/schemas/gstack.ts - Source schema:
   ```typescript
   import { z } from 'zod';
   
   export const GstackFrontmatter = z.object({
     name: z.string().min(1),
     'preamble-tier': z.number().int().min(1).max(4),
     version: z.string().optional(),
     description: z.string(),
     'allowed-tools': z.array(z.string()),
   }).strict(); // Fail on unknown fields per CONTEXT.md
   
   export type GstackFrontmatter = z.infer<typeof GstackFrontmatter>;
   ```

2. src/schemas/copilot.ts - Target schema:
   ```typescript
   import { z } from 'zod';
   
   export const CopilotFrontmatter = z.object({
     name: z.string().min(1),
     description: z.string(),
     'argument-hint': z.string().optional(),
     'allowed-tools': z.string(), // comma-separated
   }).strict();
   
   export type CopilotFrontmatter = z.infer<typeof CopilotFrontmatter>;
   ```

3. src/schemas/index.ts - Re-export both schemas

4. tests/schemas.test.ts - Test cases:
   - Valid gstack frontmatter parses
   - Missing required field throws
   - Unknown field throws (strict mode)
   - preamble-tier out of range throws
   - Valid Copilot frontmatter parses
   - allowed-tools type mismatch throws
  </action>
  <verify>
    <automated>npx vitest run tests/schemas.test.ts --reporter=verbose</automated>
  </verify>
  <done>Both schemas export correctly, all 6+ test cases pass, strict mode rejects unknown fields</done>
